 animation css속성의 경우 vscode에서 설치한 live server라는 프로그램이랑 충돌이 있기 때문에 
 animation 구문을 사용할 때는 live server를 이용해서 웹사이트를 출력하지 않는게 좋습니다.

 :transition 속성과 비슷하짐난 hover, active 구문 없이 스스로 애니메이션을 동작시켜주는 css속성의

@keyframes 원하는 이름{
    %값{css속성}
}
선택자 {animation: 원하는이름 시간s}
----------------------------------------------
animation-iteration-count : 원하는 값
입력한 값만큼 애니메이션을 반복시켜주는 css속성
값을 infinite를 넣어주면 무한하게 반복 시켜줄 수 있다

★animation: 이름 시간 횟수
시간 옆에 원하는 숫자를 작성하면 iteration-count 대신 그 역할을 수행 할 수 있다
----------------------------------------------
animation-delay: ?s
입력해준 시간값 만큼의 시간이 흐른 다음에 애니메이션 실행
transition-delay와 기능은 동일하지만 차이점은 애니메이션이 동작후 다시 원상태로 돌아올때
transition-delay와 다르게 지연시간 없이 바로 돌아온다는 차이점이 있다.

animation-timing-function
애니메이션 효과를 변경해주는 속성으로 transition-timing-function과 옵션이 동일하다
-----------------------------------------------
@keyframes 원하는 이름{
    20%{
        css속성
    }
애니메이션 시간이 10초기존 처음 태가기 있던 장소에서 목적지 까지 도달하는데 2초
    50%{
        css속성
    }
다시 원래지점으로 돌아오는데 5초
}
100%일 경우 0초만에 원래지점으로 돌아오는건 불가능 하기 때문에 태그가 목적지에 도착하는 순간
다시 원래지점으로 돌아온다.
